---

### 问题1：Transformer 中的 softmax 计算为什么需要除以 $\sqrt{d_k}$？

Softmax 的输入是 $QK^T$ 的点积结果。当 $d_k$（key 的维度）较大时，点积结果的方差会变大（约为 $d_k$），导致数值分布差异悬殊——大值经过 softmax 后趋近于 1，小值趋近于 0，**梯度几乎消失**，模型难以学习。

除以 $\sqrt{d_k}$ 起到**缩放**作用，将点积方差控制回 1 左右，让 softmax 输出更平滑、梯度更稳定。这就是论文中说的 **Scaled Dot-Product Attention**。

---

### 问题2：Transformer 中 attention score 计算时如何 mask 掉 padding 位置？

在计算 $QK^T$ 得到 attention score 矩阵后、做 softmax 之前，将 padding 位置对应的 score 设为**一个极大的负数**（如 $-\infty$ 或 $-10^9$）。这样经过 softmax 后，该位置的权重趋近于 0，相当于被"屏蔽"掉了。

具体做法是用一个 mask 矩阵标记 padding 位置（值为 1），然后：
$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + \text{mask}\right)V$$
其中 mask 在 padding 处为 $-\infty$，其余为 0。

---

### 问题3 / 问题8：为什么 Transformer 中加入了 positional embedding？

Transformer 的核心操作是 Self-Attention，而 Attention 机制本身是**对位置不敏感的**——它计算的是任意两个 token 之间的加权关系，无论 token 在序列的第几个位置，计算方式都一样。

但语言是有顺序的，「我爱你」和「你爱我」含义完全不同。因此需要显式地注入位置信息，让模型知道每个 token 在序列中的位置。Positional Embedding 就是给每个位置一个独特的编码向量，加到 token embedding 上，使模型具备感知顺序的能力。

---

### 问题4 / 问题10：BERT 预训练时 mask 的比例，可以 mask 更大的比例吗？

BERT 论文中 mask 的比例是 **15%**（对选中的 token，80% 替换为 `[MASK]`，10% 换为随机词，10% 保持不变）。

**不建议 mask 过大比例（如 >80%）**，原因：

1. **上下文信息严重不足**：mask 太多词后，模型几乎无法从上下文中推断被遮住的词，预训练任务变得不可解，学不到有效表示。
2. **数据利用率低**：mask 掉的词不参与损失计算，大量数据被浪费。
3. **预训练-微调不一致**：微调时没有 `[MASK]` token，mask 比例过大会加剧这种 gap。

实验表明，15% 左右是一个较好的平衡点。

### 问题5：BERT 如何进行 tokenize 操作？有什么好处？

BERT 使用 **WordPiece** 分词算法，词表大小约 30,000。

**流程：**
- 将文本切分为子词单元（subword），如 `"playing"` → `["play", "##ing"]`
- 用 `##` 前缀标记非词首的子词片段
- 添加特殊 token：`[CLS]`（句首）、`[SEP]`（分隔句）

**好处：**
- **解决 OOV（未登录词）问题**：任何生僻词都可以拆成已知子词，不会出现 `<unk>`
- **词表紧凑**：3 万规模的词表就能覆盖绝大多数语言现象
- **共享语义**：`"playing"` 和 `"played"` 共享 `"play"` 这个子词，参数利用更高效

---

### 问题6：GPT 如何进行 tokenize 操作？和 BERT 的区别是什么？

GPT 系列使用 **BPE（Byte Pair Encoding）**，与 BERT 的 WordPiece 核心思想相似但有关键区别：

| | BERT (WordPiece) | GPT (BPE) |
|---|---|---|
| 合并策略 | 基于**语言模型概率**（选能最大化训练数据似然的合并） | 基于**频率**（选出现频次最高的相邻对合并） |
| 特殊标记 | `##` 前缀标记子词 | 无，用特殊空格处理（Ġ 等） |
| 方向 | 双向编码 | 单向（自回归），tokenize 方式兼容生成 |

BPE 更简单直观，且天然适合**生成任务**（GPT 从左到右预测），而 WordPiece 更精确但需要语言模型辅助合并决策。

---

### 问题7：BERT 模型特别大，单张 GPU 训练仅能放入 1 个 batch 时，怎么训练？

核心策略是**梯度累积（Gradient Accumulation）**：

1. 每次只给 GPU 喂 1 个 batch 的数据
2. 正常前向传播 + 反向传播，但**不立即更新参数**，而是把梯度**累加**起来
3. 重复 $N$ 次后，用累加的 $N$ 份梯度做一次参数更新

这样等效于用 `batch_size = N` 进行训练，只是速度更慢。此外还可以配合：
- **混合精度训练（FP16）**：减少显存占用
- **Gradient Checkpointing**：用计算换显存，不存储中间激活值
- **ZeRO / DeepSpeed**：分布式优化器状态分片
- **Parameter-Efficient Fine-Tuning（如 LoRA）**：只训练少量新增参数

---

### 问题9：Transformer 中的残差网络结构作用是什么？

每个子层（Self-Attention 和 FFN）都采用了残差连接：

$$\text{Output} = \text{LayerNorm}(x + \text{Sublayer}(x))$$

**作用：**

1. **缓解梯度消失**：深层网络中梯度可以直接通过残差路径回传，不会被多次乘法衰减
2. **让深层网络可训练**：即使堆叠很多层（如 12 或 24 层），模型也不会退化
3. **保留原始信息**：Attention/FFN 学到的是对输入的「修正」，而非完全重新编码，降低学习难度

---

### 问题11：BERT 预训练是如何做 mask 的？

BERT 的 **Masked Language Model（MLM）** 训练流程：

1. **随机选择 15% 的 token** 作为待预测目标
2. 对这 15% 的 token 做三种操作：
   - **80%** 替换为 `[MASK]`
   - **10%** 替换为随机词
   - **10%** 保持不变
3. 模型根据上下文预测被选中 token 的原始词

为什么要混入随机词和原词？
- 微调阶段没有 `[MASK]`，如果预训练只见过 `[MASK]`，会产生 mismatch
- 随机词让模型不完全依赖 `[MASK]` 标记，迫使它真正利用上下文信息来判断

---

### 问题11（重复）：word2vec 到 BERT 改进了什么？

| | word2vec | BERT |
|---|---|---|
| **表示粒度** | 每个词一个固定向量，无法区分多义词 | **动态上下文表示**，同一词在不同句子中向量不同 |
| **信息方向** | 单向（CBOW/Skip-gram） | **双向**，同时利用左右上下文 |
| **词表问题** | OOV 词无法表示 | **子词分词**，无 OOV 问题 |
| **任务适配** | 静态词向量，作为下游模型输入 | **预训练+微调**范式，端到端适配下游任务 |
| **层次特征** | 只输出词级别的向量 | 多层 Transformer，可提取**字符→词→句**多层特征 |

核心改进：**从「一个词一个向量」到「一个词在不同语境下有不同的向量」**。比如「苹果」在「吃苹果」和「苹果手机」中，BERT 会产生完全不同的表示，而 word2vec 只有一个固定向量。
