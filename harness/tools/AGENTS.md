# tools/ — 工具具像化

把 `concepts/` 里抽象的 Harness 概念落到具体可用的工具上。
本目录的核心问题：**哪些工具能切实降低 [6 维复杂度框架](../thinking/software-project-complexity-in-the-ai-era.md) 中的某一维，或提升"胶囊化能力"这个总杠杆？**

## 文件约定

- `00-overview.md` — 工具地图：维度 × 工具矩阵（先读）
- `harnesses/` — Agent harness 框架（Claude Code、Codex、Cursor 等）
- `auxiliary/` — 辅助工具（linter、CI、文档、测试、依赖分析等）

每个工具页结构（建议，非强制）：

1. **一句话定位** —— 它是什么
2. **降低哪些维度的复杂度** —— 引用 6 维框架的具体维度名 + 是否提升"胶囊化能力"
3. **实测体验** —— 用过的真实场景、生效边界、失效场景
4. **搭配建议** —— 与本目录其他工具的组合
5. **替代方案** —— 同类竞品对比（如有）

## 收录标准

- **自己实际用过** —— 不收"听说很好"的工具
- 或在 [`practice/`](../practice/) 中验证过
- 拒绝"工具大全"式罗列——本目录不是 awesome-list，是带主张的杠杆库

## 与其他目录的关系

- `concepts/` 提出**该做什么**
- `tools/` 提供**用什么做**
- `practice/` 给出**实际做的过程与结果**

## 下一步

读完 [00-overview.md](00-overview.md) 后，按维度反查工具，按工具反查实践。
